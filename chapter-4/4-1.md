# 4.1 ภาพรวมระบบหน่วยความจำคอมพิวเตอร์

## ลักษณะของระบบหน่วยความจำ

หัวข้อหน่วยความจำคอมพิวเตอร์ที่ซับซ้อนจะเข้าใจได้ง่ายขึ้นหากเราจำแนกหน่วยความจำตามลักษณะสำคัญ ลักษณะที่สำคัญที่สุดเหล่านี้แสดงไว้ใน **ตาราง 4.1**

คำว่า **location** ในตารางหมายถึงหน่วยความจำที่อยู่ภายในหรือภายนอกคอมพิวเตอร์ หน่วยความจำภายในมักเทียบเท่ากับ **main memory** แต่ยังมีรูปแบบอื่นของหน่วยความจำภายใน โปรเซสเซอร์ต้องการหน่วยความจำท้องถิ่นของตัวเองในรูปแบบ **registers** (ดูรูปที่ 2.3) นอกจากนี้ control unit ของโปรเซสเซอร์อาจต้องการหน่วยความจำภายในของตัวเองด้วย เราจะอภิปรายประเภทหน่วยความจำภายในสองประเภทนี้ในบทต่อไป **Cache** เป็นอีกรูปแบบหนึ่งของหน่วยความจำภายใน **External memory** ประกอบด้วยอุปกรณ์เก็บข้อมูลรอบข้าง เช่น ดิสก์และเทป ที่โปรเซสเซอร์สามารถเข้าถึงได้ผ่าน **I/O controllers**

ลักษณะที่ชัดเจนของหน่วยความจำคือ **ความจุ** สำหรับหน่วยความจำภายในจะแสดงเป็น **bytes** (1 byte = 8 bits) หรือ **words** ความยาว word ทั่วไปคือ 8, 16 และ 32 bits ความจุหน่วยความจำภายนอกแสดงเป็น **bytes**

## หน่วยถ่ายโอนข้อมูล (Unit of Transfer)

สำหรับหน่วยความจำภายใน หน่วยถ่ายโอนเท่ากับจำนวนเส้นไฟฟ้าที่เข้าและออกจากโมดูลหน่วยความจำ อาจเท่ากับความยาว word แต่ส่วนใหญ่จะใหญ่กว่า เช่น 64, 128 หรือ 256 bits

### แนวคิดที่เกี่ยวข้อง 3 ประการสำหรับหน่วยความจำภายใน:

**Word**: หน่วยองค์กรตามธรรมชาติของหน่วยความจำ ขนาด word มักเท่ากับจำนวน bits ที่ใช้แสดง integer และความยาวคำสั่ง แม้จะมีข้อยกเว้นมากมาย เช่น CRAY C90 มี word 64-bit แต่ใช้ integer 46-bit, Intel x86 มีความยาวคำสั่งหลากหลาย

**Addressable units**: ระบบบางระบบใช้ word เป็นหน่วยที่สามารถ addressing ได้ แต่ระบบหลายระบบอนุญาต addressing ที่ระดับ byte สัมพันธ์ระหว่างความยาว address A bits กับจำนวน N ของ addressable units คือ **2^A = N**

**Unit of transfer**: สำหรับ main memory คือจำนวน bits ที่อ่านหรือเขียนพร้อมกัน ไม่จำเป็นต้องเท่ากับ word หรือ addressable unit สำหรับ external memory ข้อมูลถ่ายโอนในหน่วยใหญ่กว่าเรียกว่า **blocks**

## วิธีการเข้าถึงข้อมูล

**Sequential access**: จัดระเบียบเป็น records ต้องเข้าถึงตามลำดับเชิงเส้น ใช้ addressing information ในการแยก records และช่วยในการดึงข้อมูล

**Direct access**: แต่ละ block/record มี unique address ตาม physical location เข้าถึงโดยการไปยังตำแหน่งทั่วไปแล้วค้นหาแบบ sequential

**Random access**: แต่ละ location มี unique physically-wired addressing mechanism เวลาเข้าถึงคงที่ ไม่ขึ้นกับลำดับการเข้าถึงก่อนหน้า

**Associative**: random access ที่เปรียบเทียบ bit locations ภายใน word พร้อมกันเพื่อหาค่า match ดึง word ตามเนื้อหาแทน address

## พารามิเตอร์ประสิทธิภาพ

จากมุมมองผู้ใช้ ลักษณะสำคัญ 2 ประการคือ **ความจุ** และ **ประสิทธิภาพ** พารามิเตอร์ประสิทธิภาพ 3 ตัว:

**Access time (latency)**: เวลาที่ใช้ในการ read/write จากการนำเสนอ address จนข้อมูลพร้อมใช้งาน

**Memory cycle time**: access time + เวลาเพิ่มเติมก่อน access ถัดไป (สำหรับ random-access memory)

**Transfer rate**: อัตราการถ่ายโอนข้อมูล = 1/cycle time (สำหรับ random-access memory)

## ลักษณะทางกายภาพ

**Volatile memory**: ข้อมูลสูญหายเมื่อไฟดับ  
**Nonvolatile memory**: ข้อมูลคงอยู่โดยไม่เสื่อมสภาพ  
**Noneraseable memory**: ไม่สามารถแก้ไขได้ (ROM)

## Memory Hierarchy

ข้อจำกัดการออกแบบหน่วยความจำคอมพิวเตอร์สรุปได้ 3 คำถาม: **How much? How fast? How expensive?**

**Trade-off**:
- Faster access time → greater cost per bit
- Greater capacity → smaller cost per bit  
- Greater capacity → slower access time

**วิธีแก้ปัญหา**: ใช้ **memory hierarchy** (รูปที่ 4.1)

**คุณสมบัติเมื่อลงสู่ระดับล่าง**:
- ลด cost per bit
- เพิ่ม capacity  
- เพิ่ม access time
- ลด frequency of access

**หลักการ Locality of Reference**: memory references โดยโปรเซสเซอร์มีแนวโน้ม cluster กัน
