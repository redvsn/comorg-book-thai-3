## 7.5 การเข้าถึงหน่วยความจำโดยตรง (Direct Memory Access)

### ข้อเสียของ Programmed และ Interrupt-Driven I/O

ทั้ง Programmed I/O และ Interrupt-Driven I/O มีข้อเสียโดยธรรมชาติสองประการ:

1. อัตราการถ่ายโอน I/O ถูกจำกัดโดยความเร็วที่โปรเซสเซอร์สามารถให้บริการอุปกรณ์ได้
2. โปรเซสเซอร์ถูกผูกมัดกับการจัดการการถ่ายโอน I/O และต้องดำเนินการคำสั่งจำนวนมากสำหรับการถ่ายโอน I/O แต่ละครั้ง

เมื่อต้องย้ายข้อมูลปริมาณมาก จำเป็นต้องใช้เทคนิคที่มีประสิทธิภาพมากขึ้น นั่นคือ **Direct Memory Access (DMA)**

### หน้าที่ของ DMA

DMA เกี่ยวข้องกับโมดูลเพิ่มเติมบนบัสระบบ โมดูล DMA สามารถเลียนแบบโปรเซสเซอร์และเข้าควบคุมระบบจากโปรเซสเซอร์ได้ โมดูล DMA จำเป็นต้องใช้บัสเฉพาะเมื่อโปรเซสเซอร์ไม่ต้องการ หรือต้องบังคับให้โปรเซสเซอร์หยุดการทำงานชั่วคราว เทคนิคหลังนี้พบได้บ่อยกว่าและเรียกว่า **Cycle Stealing** เนื่องจากโมดูล DMA จะขโมยรอบบัส

เมื่อโปรเซสเซอร์ต้องการอ่านหรือเขียนบล็อกข้อมูล จะออกคำสั่งไปยังโมดูล DMA โดยส่งข้อมูลต่อไปนี้:

- ว่าขอให้อ่านหรือเขียน โดยใช้สายควบคุมอ่านหรือเขียนระหว่างโปรเซสเซอร์กับโมดูล DMA
- ที่อยู่ของอุปกรณ์ I/O ที่เกี่ยวข้อง สื่อสารบนสายข้อมูล
- ตำแหน่งเริ่มต้นในหน่วยความจำที่จะอ่านจากหรือเขียนไปยัง สื่อสารบนสายข้อมูลและเก็บไว้ในรีจิสเตอร์ที่อยู่ของโมดูล DMA
- จำนวนคำที่จะอ่านหรือเขียน สื่อสารผ่านสายข้อมูลและเก็บไว้ในรีจิสเตอร์นับข้อมูล

จากนั้นโปรเซสเซอร์จะดำเนินงานอื่นต่อ โมดูล DMA จะถ่ายโอนบล็อกข้อมูลทั้งหมดทีละคำ โดยตรงจากหรือไปยังหน่วยความจำ โดยไม่ต้องผ่านโปรเซสเซอร์ เมื่อการถ่ายโอนเสร็จสมบูรณ์ โมดูล DMA จะส่งสัญญาณอินเทอร์รัปต์ไปยังโปรเซสเซอร์ ดังนั้น โปรเซสเซอร์จะเข้ามาเกี่ยวข้องเฉพาะที่จุดเริ่มต้นและจุดสิ้นสุดของการถ่ายโอนเท่านั้น

### การกำหนดค่า DMA

กลไก DMA สามารถกำหนดค่าได้หลายวิธี:

- **(ก) บัสเดียว DMA แยก:** โมดูลทั้งหมดใช้บัสระบบเดียวกัน โมดูล DMA ทำหน้าที่เป็นโปรเซสเซอร์แทน ใช้ Programmed I/O เพื่อแลกเปลี่ยนข้อมูลระหว่างหน่วยความจำกับโมดูล I/O ผ่านโมดูล DMA การกำหนดค่านี้ไม่มีประสิทธิภาพเนื่องจากการถ่ายโอนแต่ละคำใช้รอบบัสสองรอบ
- **(ข) บัสเดียว DMA-I/O รวม:** มีเส้นทางระหว่างโมดูล DMA กับโมดูล I/O หนึ่งตัวหรือมากกว่าที่ไม่รวมบัสระบบ สามารถลดจำนวนรอบบัสที่จำเป็นได้อย่างมาก
- **(ค) บัส I/O:** เชื่อมต่อโมดูล I/O กับโมดูล DMA โดยใช้บัส I/O ซึ่งลดจำนวนอินเทอร์เฟซ I/O ในโมดูล DMA ให้เหลือหนึ่งตัวและให้การกำหนดค่าที่ขยายได้ง่าย

### ตัวควบคุม DMA Intel 8237A

ตัวควบคุม DMA Intel 8237A เชื่อมต่อกับตระกูลโปรเซสเซอร์ 80x86 และหน่วยความจำ DRAM เพื่อให้ความสามารถ DMA เมื่อโมดูล DMA ต้องการใช้บัสระบบ จะส่งสัญญาณที่เรียกว่า HOLD ไปยังโปรเซสเซอร์ โปรเซสเซอร์ตอบสนองด้วยสัญญาณ HLDA (Hold Acknowledge) ซึ่งบ่งชี้ว่าโมดูล DMA สามารถใช้บัสได้

ขั้นตอนการถ่ายโอนบล็อกข้อมูลจากหน่วยความจำไปยังดิสก์:

1. อุปกรณ์ต่อพ่วง (เช่น ตัวควบคุมดิสก์) จะร้องขอบริการ DMA โดยดึง DREQ ให้สูง
2. DMA จะตั้ง HRQ ให้สูง ส่งสัญญาณไปยัง CPU ผ่านขา HOLD ว่าต้องการใช้บัส
3. CPU จะเสร็จสิ้นรอบบัสปัจจุบันและตอบสนองด้วยการตั้ง HLDA ให้สูง บอก DMA 8237 ว่าสามารถใช้บัสได้
4. DMA จะเปิดใช้งาน DACK บอกอุปกรณ์ต่อพ่วงว่าจะเริ่มถ่ายโอนข้อมูล
5. DMA เริ่มถ่ายโอนข้อมูลจากหน่วยความจำไปยังอุปกรณ์ต่อพ่วง โดยวางที่อยู่ของไบต์แรกบนบัสที่อยู่ เปิดใช้งาน MEMR อ่านไบต์จากหน่วยความจำเข้าบัสข้อมูล แล้วเปิดใช้งาน IOW เพื่อเขียนไปยังอุปกรณ์ต่อพ่วง จากนั้น DMA ลดค่าตัวนับและเพิ่มค่าตัวชี้ที่อยู่ และทำซ้ำจนกว่าจำนวนนับจะถึงศูนย์
6. หลังจาก DMA เสร็จงานแล้ว จะปิดใช้งาน HRQ ส่งสัญญาณให้ CPU รู้ว่าสามารถควบคุมบัสคืนได้

8237 DMA เป็นที่รู้จักในฐานะ **Fly-By DMA Controller** หมายความว่าข้อมูลที่กำลังย้ายไม่ผ่านชิป DMA และไม่ถูกเก็บไว้ในชิป DMA ดังนั้น DMA สามารถถ่ายโอนข้อมูลระหว่างพอร์ต I/O กับที่อยู่หน่วยความจำเท่านั้น ไม่ใช่ระหว่างพอร์ต I/O สองพอร์ตหรือสองตำแหน่งหน่วยความจำ

8237 มีช่อง DMA สี่ช่องที่สามารถโปรแกรมได้อิสระ และมีชุดรีจิสเตอร์ควบคุม/คำสั่งห้าตัว:

- **Command:** โปรเซสเซอร์โหลดรีจิสเตอร์นี้เพื่อควบคุมการทำงานของ DMA
- **Status:** โปรเซสเซอร์อ่านรีจิสเตอร์นี้เพื่อพิจารณาสถานะ DMA
- **Mode:** โปรเซสเซอร์ตั้งค่ารีจิสเตอร์นี้เพื่อกำหนดโหมดการทำงานของ DMA
- **Single Mask:** โปรเซสเซอร์ตั้งค่ารีจิสเตอร์นี้เพื่อปิดหรือเปิดใช้งานช่องเฉพาะ
- **All Mask:** คล้ายกับ Single Mask แต่ช่องทั้งหมดสามารถปิดหรือเปิดใช้งานด้วยการดำเนินการเขียนครั้งเดียว
