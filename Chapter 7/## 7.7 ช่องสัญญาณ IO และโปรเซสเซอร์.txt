## 7.7 ช่องสัญญาณ I/O และโปรเซสเซอร์

### วิวัฒนาการของฟังก์ชัน I/O

เมื่อระบบคอมพิวเตอร์พัฒนาขึ้น มีรูปแบบของความซับซ้อนและความซับซ้อนที่เพิ่มขึ้นของส่วนประกอบแต่ละส่วน ขั้นตอนวิวัฒนาการสามารถสรุปได้ดังนี้:

1. **ขั้นที่ 1:** CPU ควบคุมอุปกรณ์ต่อพ่วงโดยตรง พบได้ในอุปกรณ์ที่ควบคุมด้วยไมโครโปรเซสเซอร์อย่างง่าย
2. **ขั้นที่ 2:** เพิ่มตัวควบคุมหรือโมดูล I/O CPU ใช้ Programmed I/O โดยไม่มีอินเทอร์รัปต์ ด้วยขั้นตอนนี้ CPU แยกออกจากรายละเอียดเฉพาะของอินเทอร์เฟซอุปกรณ์ภายนอก
3. **ขั้นที่ 3:** ใช้การกำหนดค่าเดียวกันกับขั้นที่ 2 แต่ตอนนี้ใช้อินเทอร์รัปต์ CPU ไม่จำเป็นต้องใช้เวลารอให้การดำเนินการ I/O เสร็จสิ้น ทำให้ประสิทธิภาพเพิ่มขึ้น
4. **ขั้นที่ 4:** โมดูล I/O ได้รับการเข้าถึงหน่วยความจำโดยตรงผ่าน DMA สามารถย้ายบล็อกข้อมูลไปยังหรือจากหน่วยความจำโดยไม่ต้องเกี่ยวข้องกับ CPU ยกเว้นที่จุดเริ่มต้นและจุดสิ้นสุดของการถ่ายโอน
5. **ขั้นที่ 5:** โมดูล I/O ได้รับการปรับปรุงให้เป็นโปรเซสเซอร์ในตัวเอง โดยมีชุดคำสั่งเฉพาะทางสำหรับ I/O CPU สั่งโปรเซสเซอร์ I/O ให้ดำเนินการโปรแกรม I/O ในหน่วยความจำ โปรเซสเซอร์ I/O ดึงและดำเนินการคำสั่งเหล่านี้โดยไม่ต้องมี CPU เข้ามาเกี่ยวข้อง สิ่งนี้ช่วยให้ CPU ระบุลำดับของกิจกรรม I/O และถูกขัดจังหวะเฉพาะเมื่อลำดับทั้งหมดเสร็จสมบูรณ์
6. **ขั้นที่ 6:** โมดูล I/O มีหน่วยความจำเฉพาะของตัวเองและเป็นคอมพิวเตอร์ในตัวเอง สถาปัตยกรรมนี้ช่วยให้ควบคุมอุปกรณ์ I/O จำนวนมากด้วยการมีส่วนร่วมของ CPU น้อยที่สุด

เมื่อดำเนินไปตามเส้นทางวิวัฒนาการนี้ ฟังก์ชัน I/O มากขึ้นเรื่อย ๆ จะดำเนินการโดยไม่ต้องมี CPU เข้ามาเกี่ยวข้อง สำหรับขั้นที่ 5 โมดูล I/O มักเรียกว่า **ช่องสัญญาณ I/O (I/O Channel)** สำหรับขั้นที่ 6 มักใช้คำว่า **โปรเซสเซอร์ I/O (I/O Processor)**

### คุณลักษณะของช่องสัญญาณ I/O

ช่องสัญญาณ I/O แสดงถึงส่วนขยายของแนวคิด DMA ช่องสัญญาณ I/O มีความสามารถในการดำเนินการคำสั่ง I/O ซึ่งให้การควบคุมการดำเนินการ I/O อย่างสมบูรณ์ ในระบบคอมพิวเตอร์ที่มีอุปกรณ์ดังกล่าว CPU จะไม่ดำเนินการคำสั่ง I/O คำสั่งดังกล่าวถูกเก็บไว้ในหน่วยความจำหลักเพื่อดำเนินการโดยโปรเซสเซอร์เฉพาะทางในช่องสัญญาณ I/O เอง

**ช่องสัญญาณ I/O มีสองประเภท:**

- **Selector Channel:** ควบคุมอุปกรณ์ความเร็วสูงหลายชิ้นและในแต่ละครั้งทุ่มเทให้กับการถ่ายโอนข้อมูลกับอุปกรณ์เพียงชิ้นเดียว
- **Multiplexor Channel:** สามารถจัดการ I/O กับอุปกรณ์หลายชิ้นในเวลาเดียวกัน สำหรับอุปกรณ์ความเร็วต่ำ **Byte Multiplexor** จะรับหรือส่งอักขระโดยเร็วที่สุดเท่าที่จะทำได้ไปยังอุปกรณ์หลายชิ้น สำหรับอุปกรณ์ความเร็วสูง **Block Multiplexor** จะสลับบล็อกข้อมูลจากอุปกรณ์หลายชิ้น
