## 7.9 โครงสร้าง I/O ของ IBM zEnterprise EC12

zEnterprise EC12 เป็นข้อเสนอคอมพิวเตอร์เมนเฟรมล่าสุดของ IBM ระบบนี้อิงจากการใช้ชิปโปรเซสเซอร์ zEC12 ซึ่งเป็นชิปมัลติคอร์ 5.5 GHz ที่มีหกคอร์ สถาปัตยกรรม zEC12 สามารถมีชิปโปรเซสเซอร์สูงสุด 101 ชิ้น รวมเป็น 606 คอร์

### โครงสร้างช่องสัญญาณ

zEnterprise EC12 มีระบบย่อย I/O เฉพาะที่จัดการการดำเนินการ I/O ทั้งหมด ออฟโหลดภาระการประมวลผลและหน่วยความจำนี้ออกจากโปรเซสเซอร์หลักอย่างสมบูรณ์

คอร์โปรเซสเซอร์สูงสุด 4 ตัวสามารถทุ่มเทให้กับการใช้งาน I/O โดยสร้างระบบย่อยช่องสัญญาณ (Channel Subsystem: CSS) สี่ระบบ แต่ละ CSS ประกอบด้วยองค์ประกอบต่อไปนี้:

- **System Assist Processor (SAP):** SAP คือคอร์โปรเซสเซอร์ที่กำหนดค่าสำหรับการดำเนินการ I/O บทบาทของมันคือการออฟโหลดการดำเนินการ I/O และจัดการช่องสัญญาณและคิวการดำเนินการ I/O ช่วยให้โปรเซสเซอร์อื่น ๆ ทุ่มเทให้กับตรรกะแอปพลิเคชันได้
- **Hardware System Area (HSA):** HSA คือส่วนสงวนของหน่วยความจำระบบที่มีการกำหนดค่า I/O ถูกใช้โดย SAP สงวนพื้นที่คงที่ 32 GB ซึ่งไม่ใช่ส่วนหนึ่งของหน่วยความจำที่ลูกค้าซื้อ
- **Logical Partitions (LP):** Logical Partition เป็นรูปแบบของเครื่องเสมือน แต่ละ CSS รองรับ Logical Partition ได้สูงสุด 16 Partition
- **Subchannels:** Subchannel ปรากฏต่อโปรแกรมในฐานะอุปกรณ์ลอจิก มี Subchannel หนึ่งตัวสำหรับแต่ละอุปกรณ์ I/O ที่ CSS สามารถระบุที่อยู่ได้ รองรับ Subchannel สูงสุด 196k ต่อ CSS
- **Channel Path:** Channel Path คืออินเทอร์เฟซเดียวระหว่าง Channel Subsystem กับหน่วยควบคุมหนึ่งหน่วยหรือมากกว่าผ่านช่องสัญญาณ แต่ละ CSS สามารถมี Channel Path ได้สูงสุด 256 เส้นทาง
- **Channel:** ช่องสัญญาณเป็นโปรเซสเซอร์ขนาดเล็กที่สื่อสารกับหน่วยควบคุม I/O (Control Units: CU) จัดการการถ่ายโอนข้อมูลระหว่างหน่วยความจำกับอุปกรณ์ภายนอก

### การจัดระเบียบระบบ I/O

ระบบ Enterprise EC12 ประกอบด้วยกรอบ (Frame) ขนาดใหญ่สองอัน:

**คุณสมบัติทางกายภาพ:**
- น้ำหนัก: 2,430 กก. (5,358 ปอนด์)
- ความกว้าง: 1.568 ม. (5.14 ฟุต)
- ความลึก: 1.69 ม. (6.13 ฟุต)
- ความสูง: 2.015 ม. (6.6 ฟุต)

**A Frame (กรอบขวา):** ประกอบด้วยสองชั้นวาง ชั้นบนเป็นชั้นวางโปรเซสเซอร์ที่มีสี่สล็อตสำหรับรองรับ Processor Books สูงสุดสี่เล่มที่เชื่อมต่อกันอย่างสมบูรณ์ แต่ละเล่มประกอบด้วย Multichip Module (MCM) การ์ดหน่วยความจำ และการเชื่อมต่อชั้นวาง ชั้นล่างเป็นชั้นวาง I/O ที่มีฮาร์ดแวร์ I/O รวมถึงมัลติเพล็กเซอร์และช่องสัญญาณ

**Z Frame (กรอบซ้าย):** มีแบตเตอรี่ภายในและแหล่งจ่ายไฟ รวมถึงพื้นที่สำหรับองค์ประกอบสนับสนุนหนึ่งตัวหรือมากกว่า

### การเชื่อมต่อ I/O

Processor Book แต่ละเล่มรองรับโครงสร้างพื้นฐาน I/O ภายในสองแบบ:
- **InfiniBand:** สำหรับ I/O Cage และ I/O Drawer
- **PCI Express (PCIe):** สำหรับ I/O Drawer

การเชื่อมต่อ InfiniBand จาก Processor Book ไปยัง I/O Cages และ I/O Drawers ผ่าน Host Channel Adapter (HCA) Fanout ซึ่งมีลิงค์ InfiniBand ไปยัง InfiniBand Multiplexor ใน I/O Cage หรือ Drawer

InfiniBand Multiplexor รองรับเทคโนโลยี I/O อื่น ๆ:
- **ESCON (Enterprise Systems Connection):** รองรับการเชื่อมต่อกับดิสก์ เทป และอุปกรณ์เครื่องพิมพ์โดยใช้เทคโนโลยีแบบ Fiber ที่เป็นกรรมสิทธิ์
- **Ethernet:** ให้การเชื่อมต่อ 1 Gbps และ 10 Gbps กับอุปกรณ์ที่รองรับเทคโนโลยีเครือข่ายท้องถิ่นนี้

การเชื่อมต่อ PCIe จาก Processor Book ไปยัง I/O Drawers ผ่าน PCIe Fanout ไปยัง PCIe Switches PCIe Switches สามารถเชื่อมต่อกับตัวควบคุมอุปกรณ์จำนวนหนึ่ง

แต่ละ Book มีการรวมกันของ InfiniBand HCA และ PCIe Fanouts สูงสุด 8 ตัว แต่ละ Fanout รองรับการเชื่อมต่อสูงสุด 32 รายการ รวมเป็นจำนวนการเชื่อมต่อสูงสุดต่อ Processor Book แต่ละการเชื่อมต่อควบคุมโดยโปรเซสเซอร์ช่องสัญญาณ
