## 7.3 Programmed I/O

เทคนิคที่เป็นไปได้สามวิธีสำหรับการดำเนินการ I/O ได้แก่:

| วิธีการ | ไม่ใช้อินเทอร์รัปต์ | ใช้อินเทอร์รัปต์ |
|---|---|---|
| ถ่ายโอน I/O ไปยังหน่วยความจำผ่านโปรเซสเซอร์ | Programmed I/O | Interrupt-Driven I/O |
| ถ่ายโอน I/O ไปยังหน่วยความจำโดยตรง | Direct Memory Access (DMA) | — |

**Programmed I/O:** ข้อมูลถูกแลกเปลี่ยนระหว่างโปรเซสเซอร์กับโมดูล I/O โปรเซสเซอร์ดำเนินการโปรแกรมที่ให้การควบคุมโดยตรงต่อการดำเนินการ I/O รวมถึงการตรวจสอบสถานะอุปกรณ์ การส่งคำสั่งอ่านหรือเขียน และการถ่ายโอนข้อมูล เมื่อโปรเซสเซอร์ออกคำสั่งไปยังโมดูล I/O จะต้องรอจนกว่าการดำเนินการ I/O จะเสร็จสมบูรณ์ หากโปรเซสเซอร์เร็วกว่าโมดูล I/O นี่ถือเป็นการสูญเสียเวลาโปรเซสเซอร์

**Interrupt-Driven I/O:** โปรเซสเซอร์ออกคำสั่ง I/O แล้วดำเนินการคำสั่งอื่นต่อไป และถูกขัดจังหวะโดยโมดูล I/O เมื่อโมดูลดำเนินงานเสร็จสิ้น

**Direct Memory Access (DMA):** โมดูล I/O และหน่วยความจำหลักแลกเปลี่ยนข้อมูลโดยตรงโดยไม่ต้องผ่านโปรเซสเซอร์

### ภาพรวมของ Programmed I/O

เมื่อโปรเซสเซอร์กำลังดำเนินการโปรแกรมและพบคำสั่งที่เกี่ยวข้องกับ I/O จะดำเนินการคำสั่งนั้นโดยออกคำสั่งไปยังโมดูล I/O ที่เหมาะสม ด้วย Programmed I/O โมดูล I/O จะดำเนินการที่ร้องขอแล้วตั้งบิตที่เหมาะสมในรีจิสเตอร์สถานะ I/O โมดูล I/O จะไม่ดำเนินการเพิ่มเติมเพื่อแจ้งเตือนโปรเซสเซอร์ ดังนั้น โปรเซสเซอร์มีหน้าที่ตรวจสอบสถานะของโมดูล I/O เป็นระยะจนกว่าจะพบว่าการดำเนินการเสร็จสมบูรณ์

### คำสั่ง I/O

ในการดำเนินการคำสั่งที่เกี่ยวข้องกับ I/O โปรเซสเซอร์จะออกที่อยู่ที่ระบุโมดูล I/O และอุปกรณ์ภายนอกเฉพาะ พร้อมทั้งคำสั่ง I/O คำสั่ง I/O มีสี่ประเภทที่โมดูล I/O อาจได้รับเมื่อถูกระบุที่อยู่โดยโปรเซสเซอร์:

- **ควบคุม (Control):** ใช้เพื่อเปิดใช้งานอุปกรณ์ต่อพ่วงและบอกให้ดำเนินการ ตัวอย่างเช่น เครื่องเทปแม่เหล็กอาจได้รับคำสั่งให้กรอกลับหรือเลื่อนไปข้างหน้าหนึ่งระเบียน คำสั่งเหล่านี้ถูกปรับให้เหมาะกับอุปกรณ์ต่อพ่วงแต่ละประเภท
- **ทดสอบ (Test):** ใช้เพื่อทดสอบสภาวะสถานะต่าง ๆ ที่เกี่ยวข้องกับโมดูล I/O และอุปกรณ์ต่อพ่วง โปรเซสเซอร์ต้องการทราบว่าอุปกรณ์ที่สนใจเปิดอยู่และพร้อมใช้งาน รวมถึงต้องการทราบว่าการดำเนินการ I/O ล่าสุดเสร็จสมบูรณ์หรือมีข้อผิดพลาดเกิดขึ้นหรือไม่
- **อ่าน (Read):** ทำให้โมดูล I/O รับรายการข้อมูลจากอุปกรณ์ต่อพ่วงและวางไว้ในบัฟเฟอร์ภายใน จากนั้นโปรเซสเซอร์สามารถรับรายการข้อมูลได้โดยร้องขอให้โมดูล I/O วางไว้บนบัสข้อมูล
- **เขียน (Write):** ทำให้โมดูล I/O รับรายการข้อมูล (ไบต์หรือคำ) จากบัสข้อมูล แล้วส่งรายการข้อมูลนั้นไปยังอุปกรณ์ต่อพ่วง

### คำสั่ง I/O ในเชิงโปรแกรม

ด้วย Programmed I/O มีความสอดคล้องกันอย่างใกล้ชิดระหว่างคำสั่งที่เกี่ยวข้องกับ I/O ที่โปรเซสเซอร์ดึงจากหน่วยความจำกับคำสั่ง I/O ที่โปรเซสเซอร์ออกไปยังโมดูล I/O เพื่อดำเนินการตามคำสั่ง รูปแบบของคำสั่งขึ้นอยู่กับวิธีที่อุปกรณ์ภายนอกถูกกำหนดที่อยู่

โดยทั่วไปจะมีอุปกรณ์ I/O จำนวนมากที่เชื่อมต่อผ่านโมดูล I/O กับระบบ แต่ละอุปกรณ์ได้รับตัวระบุเฉพาะหรือที่อยู่ เมื่อโปรเซสเซอร์ออกคำสั่ง I/O คำสั่งจะประกอบด้วยที่อยู่ของอุปกรณ์ที่ต้องการ ดังนั้นแต่ละโมดูล I/O จะต้องตีความสายที่อยู่เพื่อพิจารณาว่าคำสั่งนั้นมีไว้สำหรับตัวเองหรือไม่

เมื่อโปรเซสเซอร์ หน่วยความจำหลัก และ I/O ใช้บัสร่วมกัน จะมีสองโหมดที่อยู่ที่เป็นไปได้:

**Memory-Mapped I/O:** มีพื้นที่ที่อยู่เดียวสำหรับตำแหน่งหน่วยความจำและอุปกรณ์ I/O โปรเซสเซอร์ถือว่ารีจิสเตอร์สถานะและข้อมูลของโมดูล I/O เป็นตำแหน่งหน่วยความจำ และใช้คำสั่งเครื่องเดียวกันในการเข้าถึงทั้งหน่วยความจำและอุปกรณ์ I/O ตัวอย่างเช่น ด้วยสายที่อยู่ 10 เส้น สามารถรองรับตำแหน่งหน่วยความจำและที่อยู่ I/O รวมกันทั้งหมด 2¹⁰ = 1,024 ที่อยู่ในการผสมผสานใด ๆ

**Isolated I/O:** บัสติดตั้งสายคำสั่งหน่วยความจำอ่านและเขียน รวมถึงสายคำสั่งรับเข้าและส่งออก สายคำสั่งระบุว่าที่อยู่หมายถึงตำแหน่งหน่วยความจำหรืออุปกรณ์ I/O ช่วงที่อยู่เต็มอาจใช้ได้กับทั้งสอง ด้วยสายที่อยู่ 10 เส้น ระบบอาจรองรับทั้งตำแหน่งหน่วยความจำ 1,024 แห่งและที่อยู่ I/O 1,024 แห่ง เนื่องจากพื้นที่ที่อยู่ของ I/O แยกออกจากหน่วยความจำ จึงเรียกว่า Isolated I/O

ข้อดีของ Memory-Mapped I/O คือสามารถใช้คลังคำสั่งอ้างอิงหน่วยความจำขนาดใหญ่ได้ ทำให้เขียนโปรแกรมได้มีประสิทธิภาพมากขึ้น ข้อเสียคือพื้นที่ที่อยู่หน่วยความจำอันมีค่าถูกใช้ไป ทั้ง Memory-Mapped I/O และ Isolated I/O ต่างก็ใช้งานกันทั่วไป
