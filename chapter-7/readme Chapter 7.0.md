## วัตถุประสงค์การเรียนรู้

หลังจากศึกษาบทนี้แล้ว ผู้เรียนควรสามารถ:

- อธิบายการใช้งานโมดูล I/O ในฐานะส่วนหนึ่งของสถาปัตยกรรมคอมพิวเตอร์
- เข้าใจความแตกต่างระหว่าง **Programmed I/O** และ **Interrupt-Driven I/O** รวมทั้งอภิปรายข้อดีข้อเสียของแต่ละวิธี
- นำเสนอภาพรวมการทำงานของการเข้าถึงหน่วยความจำโดยตรง (Direct Memory Access)
- นำเสนอภาพรวมการทำงานของการเข้าถึงแคชโดยตรง (Direct Cache Access)
- อธิบายหน้าที่และการใช้งานของช่องสัญญาณ I/O



## บทนำ

นอกเหนือจากโปรเซสเซอร์และชุดโมดูลหน่วยความจำแล้ว องค์ประกอบสำคัญประการที่สามของระบบคอมพิวเตอร์คือชุดโมดูล I/O โมดูลแต่ละตัวเชื่อมต่อกับบัสระบบหรือสวิตช์กลาง และทำหน้าที่ควบคุมอุปกรณ์ต่อพ่วงหนึ่งชิ้นหรือมากกว่า โมดูล I/O ไม่ใช่เพียงชุดตัวเชื่อมต่อเชิงกลที่ต่อสายอุปกรณ์เข้ากับระบบเท่านั้น แต่โมดูล I/O ยังประกอบด้วยวงจรลอจิกสำหรับทำหน้าที่สื่อสารระหว่างอุปกรณ์ต่อพ่วงกับบัส

ผู้อ่านอาจสงสัยว่าเหตุใดจึงไม่เชื่อมต่ออุปกรณ์ต่อพ่วงเข้ากับบัสระบบโดยตรง เหตุผลมีดังต่อไปนี้:

- อุปกรณ์ต่อพ่วงมีความหลากหลายมากพร้อมวิธีการทำงานที่แตกต่างกัน การฝังวงจรลอจิกที่จำเป็นไว้ในโปรเซสเซอร์เพื่อควบคุมอุปกรณ์หลากหลายประเภทนั้นเป็นสิ่งที่ไม่สามารถปฏิบัติได้จริง
- อัตราการถ่ายโอนข้อมูลของอุปกรณ์ต่อพ่วงมักช้ากว่าหน่วยความจำหรือโปรเซสเซอร์มาก ดังนั้นจึงเป็นการไม่เหมาะสมที่จะใช้บัสระบบความเร็วสูงในการสื่อสารโดยตรงกับอุปกรณ์ต่อพ่วง
- ในทางกลับกัน อัตราการถ่ายโอนข้อมูลของอุปกรณ์ต่อพ่วงบางชนิดเร็วกว่าหน่วยความจำหรือโปรเซสเซอร์ ความไม่สมดุลดังกล่าวจะก่อให้เกิดความไม่มีประสิทธิภาพหากไม่มีการจัดการที่เหมาะสม
- อุปกรณ์ต่อพ่วงมักใช้รูปแบบข้อมูลและความยาวคำที่แตกต่างจากคอมพิวเตอร์ที่เชื่อมต่ออยู่

ด้วยเหตุนี้ จึงจำเป็นต้องมีโมดูล I/O โมดูลดังกล่าวมีหน้าที่หลักสองประการ (ดูรูปที่ 7.1):

1. เชื่อมต่อกับโปรเซสเซอร์และหน่วยความจำผ่านบัสระบบหรือสวิตช์กลาง
2. เชื่อมต่อกับอุปกรณ์ต่อพ่วงหนึ่งชิ้นหรือมากกว่าผ่านสายข้อมูลที่ออกแบบเฉพาะ

บทนี้เริ่มต้นด้วยการอภิปรายโดยย่อเกี่ยวกับอุปกรณ์ภายนอก ตามด้วยภาพรวมโครงสร้างและหน้าที่ของโมดูล I/O จากนั้นจะพิจารณาวิธีการต่าง ๆ ที่ฟังก์ชัน I/O สามารถทำงานร่วมกับโปรเซสเซอร์และหน่วยความจำในฐานะอินเทอร์เฟซ I/O ภายใน ต่อมาจะตรวจสอบรายละเอียดของการเข้าถึงหน่วยความจำโดยตรงและนวัตกรรมล่าสุดอย่างการเข้าถึงแคชโดยตรง และสุดท้ายจะพิจารณาอินเทอร์เฟซ I/O ภายนอกระหว่างโมดูล I/O กับโลกภายนอก
