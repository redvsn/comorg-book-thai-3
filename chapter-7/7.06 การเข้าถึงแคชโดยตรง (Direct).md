## 7.6 การเข้าถึงแคชโดยตรง (Direct Cache Access)

DMA พิสูจน์แล้วว่าเป็นวิธีที่มีประสิทธิภาพในการเพิ่มประสิทธิภาพของ I/O กับอุปกรณ์ต่อพ่วงและการรับส่งข้อมูล I/O เครือข่าย อย่างไรก็ตาม สำหรับอัตราที่เพิ่มขึ้นอย่างมากสำหรับ I/O เครือข่าย DMA ไม่สามารถปรับขนาดให้เพียงพอกับความต้องการที่เพิ่มขึ้นได้ ความต้องการนี้มาจากการนำสวิตช์ Ethernet 10 Gbps และ 100 Gbps ไปใช้อย่างแพร่หลาย

ในส่วนนี้ เราจะแสดงให้เห็นว่าการเปิดใช้งานฟังก์ชัน I/O ให้มีการเข้าถึงแคชโดยตรงสามารถเพิ่มประสิทธิภาพได้อย่างไร เทคนิคที่รู้จักกันในชื่อ **Direct Cache Access (DCA)** ตลอดส่วนนี้ เราจะพิจารณาเฉพาะแคชที่ใกล้ที่สุดกับหน่วยความจำหลัก ซึ่งเรียกว่า **Last-Level Cache** ซึ่งในบางระบบจะเป็น L2 cache และในบางระบบจะเป็น L3 cache

### DMA ที่ใช้ Last-Level Cache ที่ใช้ร่วมกัน

ระบบมัลติคอร์สมัยใหม่ประกอบด้วยทั้งแคชที่ทุ่มเทให้กับแต่ละคอร์และระดับแคชที่ใช้ร่วมกันเพิ่มเติม ไม่ว่าจะเป็น L2 หรือ L3 ด้วยขนาดที่เพิ่มขึ้นของ Last-Level Cache ที่มีอยู่ นักออกแบบระบบได้ปรับปรุงฟังก์ชัน DMA เพื่อให้ตัวควบคุม DMA มีการเข้าถึงแคชที่ใช้ร่วมกันในลักษณะที่คล้ายกับคอร์

#### โปรเซสเซอร์มัลติคอร์ Xeon

Intel Xeon เป็นตระกูลโปรเซสเซอร์ระดับไฮเอนด์ประสิทธิภาพสูงของ Intel ใช้ในเซิร์ฟเวอร์ เวิร์กสเตชันประสิทธิภาพสูง และซูเปอร์คอมพิวเตอร์ Xeon E5-2600/4600 สามารถกำหนดค่าด้วยสูงสุดแปดคอร์บนชิปเดียว แต่ละคอร์มี L1 และ L2 cache เฉพาะ มี L3 cache ที่ใช้ร่วมกันสูงสุด 20 MB โดย L3 cache แบ่งออกเป็นส่วน (Slice) ส่วนหนึ่งเกี่ยวข้องกับแต่ละคอร์

**การทำงานของ Ring:** ระบบ Ring แบบสองทิศทางความเร็วสูงเชื่อมโยงคอร์ Last-Level Cache PCIe และตัวควบคุมหน่วยความจำแบบรวม (IMC)

1. แต่ละส่วนประกอบที่เชื่อมต่อกับ Ring สองทิศทางถือเป็น Ring Agent และใช้งาน Ring Agent Logic
2. Ring Agent ให้ความร่วมมือผ่านโปรโตคอลแบบกระจายเพื่อร้องขอและจัดสรรการเข้าถึง Ring
3. เมื่อ Agent มีข้อมูลที่จะส่ง จะเลือกทิศทาง Ring ที่ส่งผลให้เส้นทางสั้นที่สุดไปยังปลายทาง

#### การใช้ DMA กับแคช

ในการดำเนินการ DMA แบบดั้งเดิม ข้อมูลถูกแลกเปลี่ยนระหว่างหน่วยความจำหลักกับอุปกรณ์ I/O โดยใช้โครงสร้างเชื่อมต่อระบบ เช่น Bus, Ring หรือ QPI Point-to-Point Matrix เมื่อมี Last-Level Cache ขนาดใหญ่ จะมีเทคนิคที่มีประสิทธิภาพมากขึ้น สำหรับการส่งออก เมื่อตัวควบคุม I/O ออกคำขออ่าน MCH จะตรวจสอบก่อนว่าข้อมูลอยู่ใน L3 cache หรือไม่ หากเป็นเช่นนั้น MCH จะส่งข้อมูลจาก cache ไปยังตัวควบคุม I/O โดยไม่จำเป็นต้องเข้าถึงหน่วยความจำหลัก

### ปัญหาด้านประสิทธิภาพที่เกี่ยวข้องกับแคช

การรับส่งข้อมูลเครือข่ายส่งผ่านในรูปแบบของลำดับบล็อกโปรโตคอลที่เรียกว่าแพ็กเก็ต โปรโตคอลระดับต่ำที่สุดคือ Ethernet ซึ่งแต่ละบล็อกข้อมูลที่รับเข้าและส่งออกประกอบด้วยแพ็กเก็ต Ethernet ที่ประกอบด้วย IP header, TCP header และข้อมูลเพย์โหลด

เมื่อประมวลผลการรับส่งข้อมูลโปรโตคอลปริมาณมาก มีสองปัจจัยที่ทำให้ประสิทธิภาพลดลง:
1. คอร์ใช้รอบนาฬิกาอันมีค่าในการคัดลอกข้อมูลระหว่างบัฟเฟอร์ระบบและบัฟเฟอร์แอปพลิเคชัน
2. เนื่องจากความเร็วของหน่วยความจำไม่ทันกับความเร็ว CPU คอร์จึงเสียเวลารอการอ่านและเขียนหน่วยความจำ

#### ขั้นตอนการรับแพ็กเก็ตขาเข้า:

1. **แพ็กเก็ตมาถึง:** NIC รับแพ็กเก็ต Ethernet ขาเข้า ประมวลผลและลบข้อมูลควบคุม Ethernet ออก
2. **DMA:** โมดูล DMA ถ่ายโอนข้อมูลรวมถึง Packet Descriptor ไปยังหน่วยความจำหลัก และต้องยกเลิกการใช้งาน Cache Line ที่สอดคล้องกันด้วย
3. **NIC ขัดจังหวะโฮสต์:** หลังจากถ่ายโอนแพ็กเก็ตจำนวนหนึ่งแล้ว NIC จะออกอินเทอร์รัปต์ไปยังโปรเซสเซอร์โฮสต์
4. **ดึง Descriptor และ Header:** คอร์ประมวลผลอินเทอร์รัปต์ และโปรแกรมจัดการอินเทอร์รัปต์จะอ่าน Descriptor และ Header ของแพ็กเก็ตที่ได้รับ
5. **Cache Miss เกิดขึ้น:** เนื่องจากนี่คือข้อมูลใหม่ที่เข้ามา Cache Line ที่สอดคล้องกับบัฟเฟอร์ระบบที่มีข้อมูลใหม่จะถูกยกเลิก คอร์จึงต้องหยุดรอเพื่ออ่านข้อมูลจากหน่วยความจำหลักเข้าสู่แคช
6. **ประมวลผล Header:** ซอฟต์แวร์โปรโตคอลดำเนินการบนคอร์เพื่อวิเคราะห์เนื้อหาของ TCP และ IP Header
7. **ถ่ายโอน Payload:** ส่วนข้อมูลของแพ็กเก็ตถูกถ่ายโอนจากบัฟเฟอร์ระบบไปยังบัฟเฟอร์แอปพลิเคชันที่เหมาะสม

#### ขั้นตอนการส่งแพ็กเก็ตขาออก:

1. **ร้องขอการถ่ายโอนแพ็กเก็ต:** แอปพลิเคชันวางข้อมูลในบัฟเฟอร์แอปพลิเคชันและแจ้ง OS
2. **สร้างแพ็กเก็ต:** OS เรียกใช้กระบวนการ TCP/IP เพื่อสร้างแพ็กเก็ต TCP/IP สำหรับการส่ง
3. **เรียกใช้การดำเนินการส่งออก:** ใช้โปรแกรมไดรเวอร์อุปกรณ์เพื่อส่งสัญญาณโมดูล DMA ว่าพร้อมส่งออก
4. **การถ่ายโอน DMA:** โมดูล DMA อ่าน Packet Descriptor และดำเนินการถ่ายโอน DMA จากหน่วยความจำหลักหรือ Last-Level Cache ไปยัง NIC
5. **NIC ส่งสัญญาณเสร็จสมบูรณ์:** หลังการถ่ายโอนเสร็จสมบูรณ์ NIC ส่งสัญญาณไปยังไดรเวอร์บนคอร์
6. **ไดรเวอร์ปลดปล่อยบัฟเฟอร์:** ไดรเวอร์ปล่อยพื้นที่บัฟเฟอร์เพื่อนำกลับมาใช้ใหม่

### กลยุทธ์ DCA

กลยุทธ์ที่ง่ายที่สุดที่นำไปใช้เป็นต้นแบบบนโปรเซสเซอร์ Intel Xeon ระหว่างปี 2549 ถึง 2553 ใช้กับการรับส่งข้อมูลเครือข่ายขาเข้าเท่านั้น ฟังก์ชัน DCA ในตัวควบคุมหน่วยความจำส่ง Prefetch Hint ไปยังคอร์ทันทีที่ข้อมูลพร้อมใช้งานในหน่วยความจำระบบ ทำให้คอร์สามารถดึงข้อมูลล่วงหน้าจากบัฟเฟอร์ระบบได้ หลีกเลี่ยง Cache Miss

กลยุทธ์ที่สมบูรณ์กว่าคือการหลีกเลี่ยงบัฟเฟอร์ระบบในหน่วยความจำหลักทั้งหมด โดยใช้ Last-Level Cache แทนหน่วยความจำหลักเพื่อบัฟเฟอร์แพ็กเก็ตและ Descriptor ของแพ็กเก็ตขาเข้าและขาออก

### Direct Data I/O ของ Intel

**Intel Direct Data I/O (DDIO)** ถูกใช้งานบนโปรเซสเซอร์ตระกูล Xeon E5 ทั้งหมด

**สำหรับการรับแพ็กเก็ต (DDIO Input):**
1. หากเกิด Cache Hit: Cache Line จะถูกอัปเดต แต่ไม่ใช่หน่วยความจำหลัก (Write-Back Strategy) Intel เรียกสิ่งนี้ว่า Write Update
2. หากเกิด Cache Miss: การดำเนินการเขียนจะเกิดขึ้นที่ Cache Line ที่จะถูกเขียนกลับไปยังหน่วยความจำหลัก Intel เรียกสิ่งนี้ว่า Write Allocate

**สำหรับการส่งแพ็กเก็ต (DDIO Output):** โปรแกรมจัดการโปรโตคอล TCP/IP สร้างแพ็กเก็ตที่จะส่งและเก็บไว้ในพื้นที่ที่จัดสรรใน L3 Cache แต่ไม่ใช่ในหน่วยความจำหลัก การดำเนินการอ่านที่เริ่มต้นโดย NIC จะได้รับการตอบสนองด้วยข้อมูลจาก Cache โดยไม่ทำให้เกิดการขับออกไปยังหน่วยความจำหลัก

DDIO มีประสิทธิภาพมากกว่า DMA สำหรับทั้งแพ็กเก็ตขาเข้าและขาออก และสามารถรองรับอัตราการรับส่งข้อมูลแพ็กเก็ตที่สูงได้ดีกว่า
