## 7.2 โมดูล I/O

### หน้าที่ของโมดูล

หน้าที่หลักหรือข้อกำหนดของโมดูล I/O จัดอยู่ในประเภทต่อไปนี้:

- การควบคุมและการกำหนดเวลา (Control and Timing)
- การสื่อสารกับโปรเซสเซอร์ (Processor Communication)
- การสื่อสารกับอุปกรณ์ (Device Communication)
- การบัฟเฟอร์ข้อมูล (Data Buffering)
- การตรวจจับข้อผิดพลาด (Error Detection)

ในช่วงเวลาใดก็ตาม โปรเซสเซอร์อาจสื่อสารกับอุปกรณ์ภายนอกหนึ่งชิ้นหรือมากกว่าในรูปแบบที่ไม่สามารถคาดเดาได้ ขึ้นอยู่กับความต้องการของโปรแกรม ทรัพยากรภายใน เช่น หน่วยความจำหลักและบัสระบบ จะต้องถูกแบ่งปันระหว่างกิจกรรมจำนวนมาก รวมถึง I/O ของข้อมูล ดังนั้น ฟังก์ชัน I/O จึงประกอบด้วยข้อกำหนดด้านการควบคุมและการกำหนดเวลา เพื่อประสานการไหลของข้อมูลระหว่างทรัพยากรภายในกับอุปกรณ์ภายนอก

ตัวอย่างเช่น การควบคุมการถ่ายโอนข้อมูลจากอุปกรณ์ภายนอกไปยังโปรเซสเซอร์อาจประกอบด้วยขั้นตอนต่อไปนี้:

1. โปรเซสเซอร์สอบถามโมดูล I/O เพื่อตรวจสอบสถานะของอุปกรณ์ที่เชื่อมต่อ
2. โมดูล I/O ส่งคืนสถานะอุปกรณ์
3. หากอุปกรณ์ทำงานได้และพร้อมส่ง โปรเซสเซอร์จะร้องขอการถ่ายโอนข้อมูลโดยออกคำสั่งไปยังโมดูล I/O
4. โมดูล I/O รับหน่วยข้อมูล (เช่น 8 หรือ 16 บิต) จากอุปกรณ์ภายนอก
5. ข้อมูลถูกถ่ายโอนจากโมดูล I/O ไปยังโปรเซสเซอร์

หากระบบใช้บัส การปฏิสัมพันธ์แต่ละครั้งระหว่างโปรเซสเซอร์กับโมดูล I/O จะเกี่ยวข้องกับการอนุญาโตตุลาการบัส (Bus Arbitration) หนึ่งครั้งหรือมากกว่า

ตัวอย่างข้างต้นยังแสดงให้เห็นว่าโมดูล I/O จะต้องสื่อสารกับทั้งโปรเซสเซอร์และอุปกรณ์ภายนอก การสื่อสารกับโปรเซสเซอร์ประกอบด้วยสิ่งต่อไปนี้:

- **การถอดรหัสคำสั่ง (Command Decoding):** โมดูล I/O รับคำสั่งจากโปรเซสเซอร์ ซึ่งโดยทั่วไปจะส่งเป็นสัญญาณบนบัสควบคุม ตัวอย่างเช่น โมดูล I/O สำหรับดิสก์ไดรฟ์อาจรับคำสั่งต่อไปนี้: READ SECTOR, WRITE SECTOR, SEEK track number และ SCAN record ID คำสั่งสองอย่างหลังแต่ละคำสั่งประกอบด้วยพารามิเตอร์ที่ส่งมาบนบัสข้อมูล
- **ข้อมูล (Data):** ข้อมูลถูกแลกเปลี่ยนระหว่างโปรเซสเซอร์กับโมดูล I/O ผ่านบัสข้อมูล
- **การรายงานสถานะ (Status Reporting):** เนื่องจากอุปกรณ์ต่อพ่วงทำงานช้ามาก การทราบสถานะของโมดูล I/O จึงมีความสำคัญ ตัวอย่างเช่น หากโมดูล I/O ถูกร้องขอให้ส่งข้อมูลไปยังโปรเซสเซอร์ (อ่าน) อาจยังไม่พร้อมเนื่องจากยังทำงานกับคำสั่ง I/O ก่อนหน้าอยู่ สัญญาณสถานะที่พบบ่อยได้แก่ BUSY และ READY อาจมีสัญญาณอื่นที่รายงานสภาวะข้อผิดพลาดต่าง ๆ ด้วย
- **การรู้จำที่อยู่ (Address Recognition):** เช่นเดียวกับที่คำแต่ละคำในหน่วยความจำมีที่อยู่ อุปกรณ์ I/O แต่ละชิ้นก็มีที่อยู่เช่นกัน ดังนั้น โมดูล I/O จะต้องรู้จักที่อยู่เฉพาะหนึ่งที่อยู่สำหรับอุปกรณ์ต่อพ่วงแต่ละชิ้นที่ควบคุม

งานสำคัญประการหนึ่งของโมดูล I/O คือการบัฟเฟอร์ข้อมูล อัตราการถ่ายโอนเข้าและออกจากหน่วยความจำหลักหรือโปรเซสเซอร์ค่อนข้างสูง ในขณะที่อัตราดังกล่าวต่ำกว่าหลายระดับขนาดสำหรับอุปกรณ์ต่อพ่วงหลายชนิด ข้อมูลที่มาจากหน่วยความจำหลักจะถูกส่งไปยังโมดูล I/O เป็นกลุ่มอย่างรวดเร็ว ข้อมูลจะถูกบัฟเฟอร์ในโมดูล I/O แล้วจึงส่งไปยังอุปกรณ์ต่อพ่วงตามอัตราข้อมูลของอุปกรณ์นั้น ในทิศทางตรงข้าม ข้อมูลจะถูกบัฟเฟอร์เพื่อไม่ให้ผูกมัดหน่วยความจำกับการดำเนินการถ่ายโอนที่ช้า

โมดูล I/O ยังมีหน้าที่รับผิดชอบในการตรวจจับข้อผิดพลาดและรายงานข้อผิดพลาดไปยังโปรเซสเซอร์ในภายหลัง ข้อผิดพลาดประเภทหนึ่งประกอบด้วยการทำงานผิดปกติทางกลและทางไฟฟ้าที่รายงานโดยอุปกรณ์ (เช่น กระดาษติด หรือแทร็กดิสก์เสียหาย) อีกประเภทหนึ่งประกอบด้วยการเปลี่ยนแปลงโดยไม่ตั้งใจต่อรูปแบบบิตระหว่างการถ่ายโอนจากอุปกรณ์ไปยังโมดูล I/O รหัสตรวจจับข้อผิดพลาดบางรูปแบบมักใช้เพื่อตรวจจับข้อผิดพลาดในการส่ง ตัวอย่างง่าย ๆ คือการใช้บิตพาริตี (Parity Bit) กับอักขระข้อมูลแต่ละตัว ตัวอย่างเช่น รหัสอักขระ IRA ใช้พื้นที่ 7 บิตของไบต์ บิตที่แปดจะถูกกำหนดเพื่อให้จำนวน 1 ทั้งหมดในไบต์เป็นเลขคู่ (พาริตีคู่) หรือเลขคี่ (พาริตีคี่) เมื่อได้รับไบต์ โมดูล I/O จะตรวจสอบพาริตีเพื่อพิจารณาว่ามีข้อผิดพลาดเกิดขึ้นหรือไม่

### โครงสร้างของโมดูล I/O

โมดูล I/O มีความหลากหลายอย่างมากในด้านความซับซ้อนและจำนวนอุปกรณ์ภายนอกที่ควบคุม รูปที่ 7.3 แสดงไดอะแกรมบล็อกทั่วไปของโมดูล I/O โมดูลเชื่อมต่อกับส่วนที่เหลือของคอมพิวเตอร์ผ่านชุดสายสัญญาณ (เช่น สายบัสระบบ) ข้อมูลที่ถ่ายโอนเข้าและออกจากโมดูลจะถูกบัฟเฟอร์ในรีจิสเตอร์ข้อมูลหนึ่งตัวหรือมากกว่า อาจมีรีจิสเตอร์สถานะหนึ่งตัวหรือมากกว่าที่ให้ข้อมูลสถานะปัจจุบัน รีจิสเตอร์สถานะอาจทำหน้าที่เป็นรีจิสเตอร์ควบคุมด้วยเพื่อรับข้อมูลควบคุมรายละเอียดจากโปรเซสเซอร์ วงจรลอจิกภายในโมดูลปฏิสัมพันธ์กับโปรเซสเซอร์ผ่านชุดสายควบคุม โปรเซสเซอร์ใช้สายควบคุมเพื่อออกคำสั่งไปยังโมดูล I/O สายควบคุมบางส่วนอาจถูกใช้โดยโมดูล I/O เอง (สำหรับสัญญาณอนุญาโตตุลาการและสัญญาณสถานะ) โมดูลยังต้องสามารถรู้จักและสร้างที่อยู่ที่เกี่ยวข้องกับอุปกรณ์ที่ควบคุมได้ด้วย

โมดูล I/O ที่รับภาระการประมวลผลรายละเอียดส่วนใหญ่และนำเสนออินเทอร์เฟซระดับสูงกับโปรเซสเซอร์ มักเรียกว่า **ช่องสัญญาณ I/O (I/O Channel)** หรือ **โปรเซสเซอร์ I/O (I/O Processor)** โมดูล I/O ที่มีความสามารถพื้นฐานและต้องการการควบคุมอย่างละเอียด มักเรียกว่า **ตัวควบคุม I/O (I/O Controller)** หรือ **ตัวควบคุมอุปกรณ์ (Device Controller)** ตัวควบคุม I/O พบได้ทั่วไปในไมโครคอมพิวเตอร์ ในขณะที่ช่องสัญญาณ I/O ใช้ในเมนเฟรม
